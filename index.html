发表于 2020-07-12 | 分类于 专题

Leetcode 第197周赛

  • 好数对的数目：求数组中数值相等的数对数目
  • 仅含 1 的子串数：由“0”“1”构成的字符串子串中仅含“1”的数目
  • 概率最大的路径：无向图中指定起点终点，求权值之积最大的路径
  • 服务中心的最佳位置：求使得欧式距离最小的寻址问题

    前言

    好数对的数目

  • 难度：简单
  • 思路：只需要计算出各个数字出现的次数，根据次数即可求数对数目。

  • 大佬代码
  • 总结：比赛时还是时间优先，数据量不大的时候，用简单的办法。

仅含 1 的子串数

  • 难度：中等

  • 思路：找到连续为1的子串，由长度计算子串数n*(n+1)/2

  • 我的代码：用了split函数，好像不太好，还是老老实实自己统计为妙

  • 注意：此处有个陷阱，极易出BUG。

  • 我原本写的是int len = tmp[i].length();,而在计算long cnt = len*(len+1)/2;时，由于len是int型，编译器先处理了int值的溢出，然后再转化成long型，导致结果出错。

  • 大佬代码(@arignote)

    概率最大的路径

  • 大佬代码(@arignote)

  • 代码解读：使用优先队列进行深搜。每次从队列中挑选概率最大的路径进行搜索，可以确保先遍历到的节点一定是到达该节点概率最大的一条路径搜索之前，为了方便找到从某节点出发可到达的节点，将邻接表转化成了图结构(二维数组)。

  • 总结：这题考点应该就是使用优先队列进行深搜。要记一下java语言的语法。

  • 得到思路启发后，我重写的版本

  • 以客户地址中，欧式距离和最小的一家客户作为初始值，然后向坐标轴方向的四个方位尝试迁移。直至迁移距离足够小，且无法找到使得距离和更小的方向时，即为所求。

  • 求欧式距离和最小的客户的过程：以客户地址平均值作为初始地址，遍历每个客户，若找到欧式距离和更小的客户，则替代。

  • 总结：其实类似思路我也想到了，读题时第一反应就是机器学习，使用迭代法确定结果。代码也写得差不多，但是我没敢继续写下去，因为使用哪个初始值，以及是否一定会收敛到最终结果，我都没法论证，纠结下最终没能完成。其实当发现草灰蛇线，又没有更好的结果时，我就敢执行下去的。毕竟即便无法求解的代码也能让别人看到自己的思路。排名前二的大佬代码思路几乎一模一样。

斗鱼网页数值修改器

发表于 2020-06-30 | 分类于 专题

插件

斗鱼网页数值修改器

特此总结一些知识点。

popup可以直接获取background句柄，来访问background环境。

在使用localStorage做存储时，注意其保存的都是字符串，因此需要使用JSON.stringify & JSON.parse来对数据处理。

chrome自带的控制台也支持jQuery风格的选取器。但是有些许不同。
$(“tag .class #name”) 只会选取出符合选取器的第一个Dom对象。

注意此处的$ 是chrome默认的函数之一(chrome commandline api)，不同于jQuery选择器。若要选择符合选取器
的所有元素，需要使用$$，类似于jQuery的$.

此外，浏览器选取的元素返回的是Dom元素或Dom元素的数组，而jQuery返回的是jQuery对象，
当没有对应结果时，返回结果的length属性为零，以此来判断是否找到匹配元素。

在使用console调试时需注意这点。

起源于看到一张图片，内容是展示斗鱼账户上近乎离谱的鱼丸鱼翅的数据。

显然，通过修改网页源代码便可以轻松做到。

受此启发，同时为了回顾五年前学习的chrome插件编程知识。花了一上午时间进行回顾，并快速实现一个小插件验证。
chrome extension经历了一个较大的改革，官方提供了更多地demo，以及兼容原先版本的新语法。
