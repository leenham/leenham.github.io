### 使数组中所有元素相等的最小操作数 * 思路:题目描述得很复杂,但实际上数组确定,只需要求各个数到平均值的距离,然后求和即可. 读懂题意的那一刻,我就决定去找规律.最后我找到了规律. * 总结:常规带循环的方法:先求出数组平均值,然后求每个数和平均值差的绝对值.求和再除以2即为所求. 由于数组唯一,因此平均值+距离和都是和n直接相关的变量,因此找规律直接列公式也是可以的.
代码
### 两球之间的磁力 * 思路: 求最小磁力的最大值,最终结果位于一定区间内,且最小磁力呈现单调性.即随着最小磁力增大,放置方案从有到无. 而最小磁力的最大值即为临界点. 因此使用二分法. 详见代码 * 检测某个最小磁力值是否满足放置方案的逻辑: * 设最小磁力为gap.使用贪心策略,即只要满足gap,便进行放置,若最终所有的球都能找到对应的篮子,则存在放置方案,否则不存在 * 因此,第一个篮子必放球.可以证明贪心策略是有效的. * 总结: 求最小值的最大值,这类题型可以用二分法来解.
代码
### 吃掉 N 个橘子的最少天数 * 思路: 自底向上的动态规划,作为程序员,这么简单而且暴露的动态规划题目能难得到我? ..................Memory Limited Exceed......好吧,反正O(n)的方法肯定是不行的.考虑优化 * 优化思路: 仔细思考一下逻辑,自底向上会求出dp[0....n]所有的值,其中很多数据是没有必要的,因此改成自顶向下的动态规划,用哈希表结构来存储已计算出的结果. * 总结: 自顶向下这层我也想到了,我最终卡壳在求dp[n-1]的时候,我还是用了dp[n] = dp[n-1] +1,这样本质上还是O(n). * btw: 贪心策略是行不通的,即当n能整除上的时候直接除以3,即dp[n] = dp[n/3]+1 if n%3==0, [举个反例,n = 321] * 而dp[n] = min(dp[n/2]+n%2+1,dp[n/3]+n%3+1);才是更好的递推公式
